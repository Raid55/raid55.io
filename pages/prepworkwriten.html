It host all of your code to make it easy to collaborate with others and to make sure that if you throw your laptop in the river your code stays in the cloud and you can pull a version right from it. Its also a social network...kinda. You can open source projects(public) for free or keep ‘em private for a small fee of 7$/month.(this is not a paid advertisement for GitHub). It is also a portfolio for all your code so that employers can see what you have done in the past and what projects you contributed to.

  5. What is the difference between git add and git commit?

    6. Git add stages what you want to commit before you commit to it! Git commit commits the code to the branch/hidden folder and makes a version you can go back to if something happens, if that thing isn't a river or a corrupt hard drive.

  6. What is the difference between git commit and git push?

    7. Committing a change is local.(unless you're committing to a github branch) If you commit and don’t push you are not impervious to rivers. Pushing a change will take your commit and push it to a remote server(the cloud/Github/Gitbucket/whatever).
